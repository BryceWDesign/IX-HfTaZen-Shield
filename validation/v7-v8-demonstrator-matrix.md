# V7–V8 Demonstrator Matrix

## Purpose

This file is the compact comparison matrix for the V7 and V8 optional-branch truth stages.

It exists to keep both optional branches centered on the real questions:

- **V7:** Does AR-05 reduce a severe local hotspot without creating worse total architecture harm?
- **V8:** Does IR-06 extract bounded secondary value from a controlled intercepted stream without harming protection?

---

## V7 Demonstrator Set

| Demonstrator ID | Branch Type | Primary Comparison Purpose |
|---|---|---|
| V7-B0 | Passive severe-hotspot baseline | Preserve no-AR-05 comparison |
| V7-A1 | Porous-UHTC local transpiration-style relief | Primary active-relief truth test |
| V7-A2 | Refractory heat-pipe local intercept | Alternate active-relief truth test |
| V7-R1 | Simplified local heat-spreading branch | Reserve local-relief comparison |

## Ranking Categories

Use the following first-pass ranking categories when V7 or V8 is eventually executed.

### Rank A
Strong continuation candidate

### Rank B
Credible continuation candidate with caveats

### Rank C
Useful comparison result but not preferred for continuation

### Rank D
Cut or deprioritize

---

## Minimum Evaluation Questions

### For V7 demonstrators
1. Did it reduce the local hotspot burden measurably?
2. Did the benefit stay local and bounded?
3. Did it avoid creating worse oxidation or structural harm?
4. Did it preserve passive-first architecture logic?
5. Is it worth carrying into V9?

### For V8 demonstrators
1. Was the intercepted source stream clearly controlled and justified?
2. Did it produce bounded useful electric output?
3. Was residual heat accounted for honestly?
4. Did it avoid worsening protection?
5. Is it worth carrying into V9?

---

## Planned Advancement Logic

### Advance by default
- Rank A
- Rank B

### Advance only if needed for comparison continuity
- Rank C

### Cut unless a very specific reason exists
- Rank D

---

## Stage Discipline Rule

No V7 or V8 branch may advance merely because optional multifunctionality sounds impressive.

Advancement must be tied to comparative branch-stage performance and total-architecture benefit.
