# V2 Pairing Matrix

## Purpose

This file is the compact comparison matrix for the V2 oxidation-control stage.

It exists to keep the stage focused on the actual question:

> Does OX-02 improve the HTZ-01 stack logic relative to HTZ-01 alone?

## Ranking Categories

Use the following first-pass ranking categories when V2 is eventually executed.

### Rank A
Strong continuation candidate

### Rank B
Credible continuation candidate with caveats

### Rank C
Useful comparison result but not preferred for continuation

### Rank D
Cut or deprioritize

---

## Minimum Evaluation Questions

Each pairing should be judged against these questions.

1. Did the coupon remain grossly intact?
2. Did the interface remain more credible than the no-transition baseline?
3. Did the transition reduce buried damage logic?
4. Did the transition avoid obvious brittle-trap behavior?
5. Is this pairing worth carrying into V3 / V4?

---

## Planned Advancement Logic

### Advance by default
- Rank A
- Rank B

- Rank C

### Cut unless a very specific reason exists
- Rank D

---

## Stage Discipline Rule

No OX-02 variant may advance merely because the stack architecture originally wanted a buried transition.

Advancement must be tied to comparative stage performance against the HTZ-01-only baseline.
